Run a GoldBrix Node

Anyone can run a full GoldBrix node. No permission, no registration, no keys required. Your node independently verifies every block and strengthens the network.

0. One command (recommended)

Full non-pruned node + public read endpoint (/api/status, /api/utxos) in one step. Keyless — the machine never holds funds. Needs x86_64 Linux, root, ≥900GB free disk.

curl -fsSL https://raw.githubusercontent.com/GOLDBRIX-GBX/goldbrix-tools/main/run-node/install-node.sh -o install-node.sh
less install-node.sh   # read before running — verify, don't trust
sudo bash install-node.sh

The script pins the binary SHA-256 (also anchored on-chain) and installs systemd services goldbrixd, gbx-read-api, gbx-indexer. When synced, expose port 8088 behind HTTPS and submit your endpoint for nodes.json — wallets pick you up via quorum, no app rebuild.

1. Requirements

OSLinux x86_64 (Ubuntu 22.04+ / Debian 12+)
RAM2 GB minimum (4 GB recommended)
Disk10 GB free (chain grows over time)
NetworkOpen inbound TCP port 39444 (recommended)

2. Download & verify the binary

The release is built with a fully reproducible process. Anyone rebuilding from source obtains byte-identical binaries — verify, don't trust.

wget https://goldbrix.app/downloads/goldbrix-30-gbx-10-x86_64-linux-gnu.tar.gz
wget https://goldbrix.app/downloads/SHA256SUMS-v30-gbx-10.txt
sha256sum goldbrix-30-gbx-10-x86_64-linux-gnu.tar.gz
Expected SHA-256:
2f384b897fc0ee106466275c376aa192a596d0feb7537bf0158a67fc6c2c794e
If the hash does not match — STOP. Do not run the binary.

3. Install

tar -xzf goldbrix-30-gbx-10-x86_64-linux-gnu.tar.gz
sudo cp goldbrix-30-gbx-10/bin/goldbrixd goldbrix-30-gbx-10/bin/goldbrix-cli /usr/local/bin/

4. Configure

mkdir -p ~/.bitcoin
cat > ~/.bitcoin/goldbrix.conf << 'EOF'
server=1
listen=1
txindex=1
EOF

Peers are discovered automatically via DNS seed seed.goldbrix.app. No manual configuration needed.

5. Run

goldbrixd -daemon
goldbrix-cli getblockcount
goldbrix-cli getpeerinfo

Block height should grow and match goldbrix.app/gbx-node-info.

6. Run as a service (recommended)

sudo tee /etc/systemd/system/goldbrixd.service << 'EOF'
[Unit]
Description=GoldBrix Core daemon
After=network.target
[Service]
ExecStart=/usr/local/bin/goldbrixd -datadir=/root/.bitcoin
Restart=always
RestartSec=10
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl daemon-reload
sudo systemctl enable --now goldbrixd

7. Verify your node's integrity (optional)

Compare your binary hash with the network's canonical release:

sha256sum /usr/local/bin/goldbrixd
curl -s https://goldbrix.app/gbx-node-info

The binary_sha256 field shows the hash of the reproducible Guix build currently running on public seed nodes.

8. Rebuild from source (verify, don't trust)

Source: goldbrix-core (git clone, tag v30-gbx-10). Full reproducible-build instructions ship inside the source tree (contrib/guix/). Two independent builds of tag v30-gbx-10 produce identical SHA-256 sums.

9. Run the public watchtower (optional)

Independent, keyless liveness monitor. Records the network's uptime history on YOUR machine — proof the system runs without its founder.

wget https://goldbrix.app/downloads/watchtower.py
python3 watchtower.py

10. Mine to your own address (solo, non-custodial)

The public pool at stratum+tcp://goldbrix.app:3333 runs in solo non-custodial mode: the coinbase of every block pays your address directly. The pool never holds funds. Fee: 0 (code-is-law) — verify live at /pool-info.

./cpuminer -a sha256d -o stratum+tcp://goldbrix.app:3333 -u bn1youraddress.worker1 -p x

Login format: bn1address.workername. Any SHA-256d stratum miner works. Rewards mature after 100 blocks. You can also run this pool software against your own node — source: goldbrix-tools. Node source: goldbrix-core.
